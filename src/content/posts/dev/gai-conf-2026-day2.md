## 上午：企業 AI 轉型與組織管理

每場 20 分鐘，主題串接。

### 企業 AI 轉型地圖

**Happy** · 91APP 產品長

從一個具體案例看起：廣告優化師（投手）→ 透過訪談 skill 萃取知識 → 由 Agent 執行 → 廣告優化師升級為**製程功能師**。

工作形式的改變是：人從「做事」變成「監看 agent 操作」。技能本身也需要 CI/CD — skill 不是建完就不動，也要持續迭代驗證。

> Agent 需要 OKR。「下禮拜要上線」這種目標 agent 才能執行。

---

### 代理人寫作

**李慕約** · 生成式 AI 年會策展人

用 agent 「把時間買回來」，流程：初稿 → 反思 → 回饋。

關鍵洞察：

- 「刪除」= 價值選擇，不是減少工作
- 給 agent 更多脈絡，內容更充實
- 可觀測性很重要：把每個步驟的輸入輸出都記錄成 log
- 人的角色是把關、打雜、編輯 — 防止「編輯污染」（過度修改反而失去原味）

---

### 打造人人想當勇者的冒險者公會

**傑哥** · 只要有人社群顧問執行長

如何讓團隊真的用 AI，不是強迫而是讓人想用：

- **導入行為就是商品，員工就是消費者** — 站在消費者角度賣給員工
- 先讓大家覺得有趣，才會覺得「這跟我有關」
- 只要提出問題就能得到獎勵，而不是只有解決問題的人獨佔獎勵
- 先做簡單、立刻有成就感的事情
- 共用工具：透過 Hermes agent 放在一台 Mac mini，共用 Markdown 檔案

> 利益才是動力，導入 AI 不是員工的義務。

### 一個 80 人團隊在 AI 浪潮裡的真實管理故事

**Wisely** · 創智動能執行副總

**FDE（Forward Deployed Engineer）**：直接嵌入客戶團隊的工程師角色，填補「產品能做什麼」與「客戶真正需要什麼」之間的落差。技術能力要強到能自己開發，工作重心卻是「讓客戶的業務真的跑起來」。由 Palantir 普及，目前 AI 企業（如 OpenAI）大量招募此職位。

管理心法：

- 沒有乾掉的黏土，沒有傳統職能包袱 — 可以揉成新角色
- 厲害的 PM 要會讀空氣；Remote 工作需要刻意培養閱讀空氣的能力
- 強迫文字化訊息反而是優點 — 讓知識留存在組織裡
- 客戶要的不是 AI，而是有人負責讓專案成功
- 轉型不是全員適應：有人能調適，有人不行，需要提前面對

### 賣實體產品的公司，也能長成 AI Native

**王大皓** · MIXXIN 一起實驗執行長

幾個具體做法：

- Agent 主動提供方法，不是被動回答
- 用 AI 輔助帶人（mentor/coaching 場景）
- 區分多個 AI 主管角色（例如：創意 AI 主管、執行 AI 主管）
- 循環餵養 context，每週主動推播
- 節點地圖 + cost 分析，用 AI 掃描

> 小蝦米撼動大鯨魚的時代，靈活就是關鍵。

### Leading to Agent-Native Company

**Kytu Lin 林裕欽** · Dcard CEO

從個人到組織的躍升：

- 人均任務數提升 → 正循環
- 讓全公司都會用 agent，不只是開發者
- 新人的 time-to-value 大幅縮短
- 真實知識散落在公司各處 — 要建立機制讓 agent 能存取

痛點：**卡死在文化，不是技術**。

解法：

1. 建立學習循環
2. Workshop 讓人實際體驗
3. 樹立未來職涯想像
4. 角色擴張（不是消失）

資料正確性問題：用對抗性查核（adversarial verification）應對。

### 我的麻瓜 AI Agent 之路

**Peggy Lo** · 非營利組織工作者

架構選型三個考量：安全、成本、可維護性（以 OpenClaw 為例）。

**卡關的三個層面**

1. 流程面：實際執行過程中發現流程的源頭本身沒有意義，把問題丟回相關部門重新檢討
2. 行為面：真正的卡關是「本人確認」這個環節，需要人在迴圈裡
3. 原生面：無障礙網站 = agent 無礙 — 無障礙設計讓 agent 更容易操作

社會影響：讓 agent 來到弱勢者身邊，AI 契機在於處理長期沉積的問題。

### 我的程式碼終於活過來了 — 從演算藝術到數位生命體

**吳哲宇** · 墨雨互動設計 MonoLab 演算藝術家 / 創辦人

演算藝術跨入 AI 的探索。核心概念：semiont（資訊生命體）。

**知識庫的建立**

如何讓 AI 真的「認識你」？不是靠 prompt，而是靠有結構的知識庫。Taiwan.md 是他的實驗：用 AI 友善的 Markdown 格式，讓台灣人自己定義台灣，讓未來的 LLM 能從在地視角理解台灣，而非依賴外部資料的扭曲詮釋。

**模型扭曲問題**

AI 從網路學習時，如果高品質內容大多來自非在地視角，學到的就是被扭曲的版本。三種因應策略，各有侷限：

- **中性模型**（如 GPT、Claude）：表面中立，實際上因訓練資料偏差而有立場，小文化、小語言被邊緣化
- **本地模型**（Sovereign AI 方向）：用在地資料訓練自己的 LLM，成本極高，且造成 AI 生態碎片化
- **主權巴別塔**：若每個國家都各自建立主權 AI，最終形成彼此無法互通的 AI 方言，如聖經中的巴別塔情景

吳哲宇的切入點：不訓練新模型，而是以開源知識庫讓現有模型「讀懂」在地視角。掌控高品質結構化內容，就是掌控敘事主權。

### 把整個公司變成 Context

**陳泰呈 Jackle** · 卡柏蒂 CUPETIT 資訊長

AI PM 的視角：比較 3-5 人小公司與百人公司對 context 管理的差異。

具體做法：

- 收集 context，用 dashboard 一覽眾山小
- 分析成功案例的簡報，用 AI 產生模板簡報
- Demo 要有料（不是空殼）
- 模擬最壞和最好的狀況，用 AI 做兵推（war gaming）

> 一切都是 context。AI 紅利期，能建立 context 護城河的公司優先勝出。

## 詞彙速查

### LLM（Large Language Model）

大型語言模型。AI Agent 的核心推論引擎。本身不執行動作，需搭配 Harness 才能成為 agent。

### AI Agent

能自主規劃、呼叫工具、執行多步驟任務的 AI 系統。第二天的主軸是：個人用 agent 已普及，下一步是讓整個組織 agent-native 化。

### OKR（Objectives and Key Results）

目標與關鍵結果。企業目標管理框架，由 Intel 發展、Google 普及。Objective 定義「想去哪」，Key Result 定義「怎麼知道到了」。講者指出 agent 同樣需要明確的 OKR — 「下禮拜要上線」這種具體目標 agent 才能自主執行。

### CI/CD（Continuous Integration / Continuous Delivery）

持續整合與持續交付。每次程式碼提交後自動執行測試、建置、部署的流程。第二天的延伸概念：AI skill 本身也需要 CI/CD，不能建完就不動，要持續迭代驗證。

### FDE（Forward Deployed Engineer）

直接嵌入客戶團隊的工程師，填補「產品能做什麼」與「客戶真正需要什麼」的落差。技術能力要強到能自己開發，工作重心卻是「讓客戶業務真的跑起來」。由 Palantir 普及，AI 時代大量崛起（OpenAI 等公司積極招募）。

### Agent-Native

以 agent 為設計前提從頭打造的公司或系統架構，類似「Cloud-Native」的概念。區別於「AI 輔助」（enhancement）— Agent-Native 把 AI agent 視為組織運作的一等公民，而非外掛工具。只有從頭重新設計架構才能真正達到，既有公司靠補丁難以實現。

### Time-to-value

新員工（或新工具）從加入到產生實際貢獻所需的時間。Agent-Native 組織的 time-to-value 大幅縮短，因為 agent 協助轉移了組織知識，新人不需要靠人傳人就能快速上手。

### 對抗性查核（Adversarial Verification）

讓 AI 扮演「反方」，試圖反駁或找漏洞於自己的產出，以驗證正確性。相較於只讓 AI 自我確認，對抗性查核更能找出邏輯錯誤與盲點。也適用於資料正確性驗證。

### ABT（AI 數位孿生大腦）

汪志謙創辦的 AI 產品，品牌名稱 ABT。將顧問專業知識 AI 化，提供 24/7 隨時待命的 AI 戰略顧問服務。核心概念是把頂級商業大師的策略洞察規模化注入企業，讓組織能以更低成本獲得專家級建議。演講以 ABT 為例說明「提供的不是對話，而是診斷報告」的 AI 變現路徑。

### Semiont（資訊生命體）

吳哲宇在演講中提出的概念，指具有自我認知層的數位知識實體 — 不只是靜態資料庫，而是有「意識結構」的資訊有機體。他的 Taiwan.md 專案以此嘗試為台灣建立數位身份，讓 AI 能從「台灣自己定義自己」的視角理解台灣，而非依賴外部訓練資料的扭曲詮釋。

### 兵推（War Gaming）

模擬最壞與最好情境的決策練習。原為軍事術語，商業語境指在真實行動前，用 AI 或人工模擬各種對手行動與市場情境，提前找出策略漏洞。AI 能快速生成多種情境劇本，大幅降低兵推的人力成本。
